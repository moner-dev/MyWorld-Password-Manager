## Bug Reports

### Before Submitting

1. **Search existing issues** - Your bug may already be reported
2. **Use latest version** - Ensure you're on the [latest release](https://github.com/moner-dev/MyWorld-Password-Manager/releases/latest)
3. **Reproduce the bug** - Confirm it happens consistently

### What to Include

```markdown
## Bug Report

**Version:** [e.g., 2.0.0]
**OS:** [e.g., Windows 11 64-bit]

**Description:**
A clear description of what the bug is.

**Steps to Reproduce:**
1. Go to '...'
2. Click on '...'
3. See error

**Expected Behavior:**
What you expected to happen.

**Actual Behavior:**
What actually happened.

**Screenshots:**
If applicable, add screenshots.

**Additional Context:**
Any other relevant information.
```

### Bug Labels

| Label | Description |
|-------|-------------|
| `bug` | Confirmed bug |
| `critical` | Security or data loss issue |
| `ui` | Visual/interface issue |
| `performance` | Speed or memory issue |
| `crash` | Application crash |

## Feature Requests

### Before Submitting

1. **Check the roadmap** - It might already be planned
2. **Search existing requests** - Someone may have suggested it
3. **Consider the scope** - Does it fit MyWorld's goals?

### What to Include

```markdown
## Feature Request

**Summary:**
Brief description of the feature.

**Problem:**
What problem does this solve?

**Proposed Solution:**
How do you envision this working?

**Alternatives Considered:**
Other ways to solve this problem.

**Additional Context:**
Mockups, examples, or references.
```

### Feature Labels

| Label | Description |
|-------|-------------|
| `enhancement` | New feature or improvement |
| `ui-request` | Interface improvement |
| `security` | Security enhancement |
| `accessibility` | Accessibility improvement |

## Documentation

Documentation improvements are always welcome! You can help by:

| Contribution | Examples |
|--------------|----------|
| **Fix typos** | Spelling, grammar, punctuation |
| **Clarify content** | Rewrite confusing sections |
| **Add examples** | Usage examples, screenshots |
| **Translate** | Localize documentation |
| **Update info** | Keep docs current |

### Documentation Files

| File | Purpose |
|------|---------|
| `README.md` | Main project documentation |
| `CHANGELOG.md` | Version history |
| `SECURITY.md` | Security policy |
| `BUILD.md` | Installation guide |
| `CONTRIBUTING.md` | This file |

### Submitting Documentation Changes

1. Fork the repository
2. Create a branch (`docs/your-change`)
3. Make your changes
4. Submit a Pull Request

## Translations

Help make MyWorld accessible to more users by contributing translations.

### Currently Supported

| Language | Status |
|----------|--------|
| English | ✅ Complete |

### How to Contribute Translations

1. Open an issue with the `translation` label
2. Specify the language you want to add
3. We'll provide the strings file
4. Submit your translations

### Translation Guidelines

- Use formal language appropriate for a security application
- Keep technical terms consistent
- Test translations in context when possible
- Provide translations for all strings (no partial submissions)

## Security Vulnerabilities

> **Important:** Do NOT open public issues for security vulnerabilities.

### Reporting Security Issues

| Method | Contact |
|--------|---------|
| **Email** | [moner.intelligence@gmail.com](mailto:moner.intelligence@gmail.com) |
| **Subject** | `[SECURITY] Brief description` |

### What to Include

- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

### Response Timeline

| Stage | Timeframe |
|-------|-----------|
| Acknowledgment | 48 hours |
| Initial Assessment | 7 days |
| Resolution | Based on severity |

See [SECURITY.md](SECURITY.md) for full details.

## Community Guidelines

### Code of Conduct

We are committed to providing a welcoming environment. Please:

| Do | Don't |
|----|-------|
| Be respectful and inclusive | Use offensive language |
| Give constructive feedback | Make personal attacks |
| Help others learn | Dismiss questions |
| Stay on topic | Spam or self-promote |
| Assume good intentions | Troll or harass |

### Communication

| Channel | Purpose |
|---------|---------|
| **Issues** | Bug reports, feature requests |
| **Pull Requests** | Documentation changes |
| **Discussions** | General questions, ideas |
| **Email** | Security issues, private matters |
